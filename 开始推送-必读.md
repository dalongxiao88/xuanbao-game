## 如果推送失败

### 情况1：提示认证失败

**解决方法：**

1. 访问：https://github.com/settings/tokens
2. 点击 "Generate new token (classic)"
3. 勾选 `repo` 权限
4. 生成并复制token
5. 运行 `配置GitHub认证.bat`
6. 再次推送时，使用token作为密码

### 情况2：网络连接失败

**解决方法：**

- 检查网络连接
- 如果在中国大陆，可能需要配置代理或使用VPN
- 运行 `诊断Git配置.bat` 检查连接状态

### 情况3：推送被拒绝

**解决方法：**

```bash
# 在命令行中执行
git pull origin main --rebase
git push origin main
```
